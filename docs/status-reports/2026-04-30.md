### 2.2 Fixes implementados (commit `c0680f5`)

**`components/ProjectDetail.tsx` y `components/BlogPostDetail.tsx`**

Antes:
```tsx
if (!project) {
  navigate('/projects', { replace: true });
  return null;
}
```

Después:
```tsx
if (!project) {
  return <Navigate to="/projects" replace />;
}
```

El primer patrón muta el router como side-effect en el cuerpo del componente, lo que React puede ejecutar varias veces (StrictMode en dev) o producir warnings. El segundo es declarativo y es la forma idiomática en React Router v6/v7.

**Ruta 404**

- Nuevo `components/NotFound.tsx` (31 líneas): página centrada con "404" grande en serif accent + título + descripción + CTA "Volver al inicio".
- Strings bilingües (`notFoundTitle`, `notFoundDescription`, `notFoundCta`) añadidas a `AppContent.ui` en `types.ts` y a las dos copias EN/ES en `contentData.ts`.
- Ruta catch-all registrada en `App.tsx` como `<Route path="*" element={<NotFound data={data} />} />`. Cae después de las demás, así que sólo captura URLs sin match.

Verificado en navegador con Playwright:
- `/this-route-does-not-exist` → 404 con header/footer y "Volver al inicio" funcionando ✓
- `/projects/nonexistent-id` → redirección con `replace` a `/projects` ✓
- `/projects/4` → detalle del proyecto carga normal, sin warnings ni errors en consola ✓

**Script `typecheck`**

Añadido a `package.json`:

### 2.3 Diagnóstico falso positivo del ticker

Tras desplegar localmente con `npm run dev`, el usuario reportó "no se actualiza el currently building". Investigación:

1. `public/activity.json` local del 2026-04-20 — 10 días desfasado.
2. `gh run list --workflow=update-activity.yml` mostró **éxitos cada 6h**, último hoy 06:59 UTC.
3. `git log origin/main` mostró **10+ commits** de `chore(activity): refresh ticker data` que no estaban en local.

Conclusión: el componente y el workflow estaban perfectos. La rama `main` local llevaba ~10 días sin `git pull`. Resuelto con `git stash → git pull --rebase → git stash pop` (mis cambios estaban sin commitear y no tocaban `public/activity.json`, así que el rebase fue limpio).

Lección: cuando un cron-driven workflow regenera ficheros versionados, la copia local se desactualiza silenciosamente. No es un bug — es comportamiento correcto del Action ("commit if changed" sólo commitea si hay diff real).
