### 4.4 Idea diferida — Métricas dinámicas

Durante la Fase 3 el responsable preguntó cómo se actualizarán las `businessMetrics`. Hoy son hardcoded. Tres opciones consideradas:

1. **Manual** (actual) — editar el fichero cuando haya release.
2. **Build-time** — GitHub Action en el repo del proyecto que abra PR al portfolio actualizando un JSON.
3. **Runtime** — fetch desde la API de GitHub (solo cubre stars/releases/commits, no tests/LoC).

**Decisión**: posponer. Sobre-ingeniería con un solo proyecto. Se reconsiderará en una fase posterior cuando haya ≥3 proyectos con métricas vivas.
